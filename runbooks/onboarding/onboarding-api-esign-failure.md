# Runbook: onboarding-api — merchant e-sign / document-step failures

**Service:** onboarding-api
**Owning team:** Onboarding On-Call (onboarding-oncall@example.com)
**Depends on:** document-service, merchant-service, notification-service
**Severity guidance:** P1 if new-merchant onboarding is fully blocked; P2 if a subset of merchants are affected.

## Symptoms
- Merchants report "unable to e-sign the Merchant Agreement" or the onboarding flow is stuck at the document step.
- onboarding-api returns HTTP 403 (permission error) or HTTP 502 from `document-service`.
- Spike in `onboarding_documents_failed_total` in Grafana; `document-service` p99 latency above 2s.

## Likely root causes
1. **document-service degraded** — the e-sign envelope is generated by document-service; if it is timing out or returning 5xx, the onboarding-api document step fails. This is the most common cause.
2. **Stale permission cache** — onboarding-api caches merchant access permissions; after a role change the cache can serve a 403 until invalidated.
3. **merchant-db replication lag** — if the merchant record was just created on the primary but the read replica is behind, document-service cannot resolve the merchant and returns 404/502.

## Diagnosis
1. Check document-service health: `GET /document-service/healthz`; confirm it is not in the `healthy but busy` state.
2. Inspect onboarding-api logs for `permission_denied` vs `upstream_timeout` markers to distinguish a cache problem from a document-service problem.
3. Check merchant-db replica lag in Grafana (`merchant_db_replica_lag_seconds`). Lag above 30s points to cause 3.

## Resolution
1. If document-service is degraded: page Onboarding On-Call, scale document-service, and follow the document-service runbook.
2. If it is a stale permission cache: invalidate the merchant's permission cache entry and retry the e-sign step. The cache TTL is 15 minutes; manual invalidation resolves it immediately.
3. If replication lag: route document-service reads to the primary until the replica catches up, then revert.

## Verification
- A test merchant can complete the e-sign step end to end.
- `onboarding_documents_failed_total` returns to baseline and document-service p99 is back under 500ms.
